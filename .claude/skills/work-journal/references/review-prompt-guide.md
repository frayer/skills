# Review Workflow Guide

## Mid-Year Review

To generate a mid-year review:

1. Make sure you have weekly notes in `journal/weekly/` covering January-June
2. Ask the skill to write your mid-year review
3. Optionally provide your company's review template — the skill will fill it in
4. If no template is provided, the skill organizes accomplishments under your objective headings

**Output:** `journal/reviews/YYYY-midyear.md`

## Year-End Review

Same process as mid-year, covering the full year (or July-December if mid-year already exists).

**Output:** `journal/reviews/YYYY-yearend.md`

## Tips for Better Reviews

- The more weekly notes you have, the better the review. Even sparse notes help.
- The skill pulls language from `journal/knowledgebase/objectives.md` — keep that file current.
- Reviews are drafted for revision. Expect to iterate 2-3 times.
